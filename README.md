# 문피아 제목 레이더

문피아 베스트 순위의 **제목만 매일 보관**하고, 제목 속 단어가 얼마나 자주·얼마나 높은 순위에 등장하는지 일간·주간·월간으로 분석하는 정적 웹사이트입니다.

## 수집 대상

- 무료 베스트
- 유료 베스트
- 선독점 베스트
- 선호작 베스트
- 베스트셀러

매일 오전 **09:10 KST**에 GitHub Actions가 각 목록의 1~30위를 수집해 `data/rankings.json`에 날짜별 스냅샷으로 저장합니다.

## 공개 사이트

- https://mikeshin0822.github.io/munpia-ranking/
- 정적 화면은 `gh-pages` 브랜치에서 게시합니다.
- 화면은 실행할 때마다 `main/data/rankings.json`의 최신 데이터를 우선 불러오므로, 매일 수집된 순위는 사이트 재배포 없이 반영됩니다.
- 원격 데이터 접근이 실패하면 게시 브랜치에 포함된 로컬 데이터로 자동 전환합니다.

## 화면

### 오늘의 제목

날짜와 베스트 종류를 선택해 제목을 순위대로 확인합니다. 작가명, 장르, 조회수 등은 제외해 제목 자체만 빠르게 볼 수 있습니다.

### 통계 · 인사이트

- 일간 / 최근 7일 / 최근 30일 분석
- 키워드 등장 횟수
- 평균 순위와 최고 순위
- 상위 순위 가중 점수 (`31 - 순위`)
- 반복되는 2어절 조합
- `미국`, `집착`처럼 원하는 단어 직접 조회

## 로컬 실행

```bash
npm test
npm run serve
```

브라우저에서 `http://localhost:4173`을 엽니다.

## 수동 수집

```bash
npm run collect
```

문피아 모바일 공개 베스트 페이지의 표지 이미지 대체 텍스트를 우선 사용해 제목을 추출합니다. 한 목록의 수집이 실패해도 다른 목록은 저장하며, 30개 미만이면 `partial` 상태로 기록합니다.

## 데이터 구조

```text
data/rankings.json
└── snapshots[]
    ├── date
    └── rankings
        ├── free_today
        ├── paid_today
        ├── exclusive_today
        ├── favorites
        └── bestseller
```

초기 데이터는 2026년 7월 22~23일 대화에서 수집한 결과를 바탕으로 넣었습니다. 당시 선호작 베스트는 확보된 10위까지만 초기 등록했으며, 첫 자동 수집부터 30위까지 채워집니다.
